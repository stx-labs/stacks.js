# Flow 2 — Update an STX-only stake

The same user extends their lock by 4 cycles, adds 10,000 STX, and rotates
to a new signer-manager — all in one call. Pass `0n` / `0` to skip a
dimension.

```ts
import { buildStakeUpdate } from '@stacks/bitcoin-staking';
import { broadcastTransaction, fetchNonce, signTransaction } from '@stacks/transactions';

const network = 'mainnet';

const tx = buildStakeUpdate({
  signerManager: 'SP000…USER.signer-manager-v2', // rotate (or pass current to keep)
  cyclesToExtend: 4,
  amountIncrease: 10_000_000_000n,
  publicKey: user.stxPublicKey,
  // TODO: check with core team if we should add a start burn ht check here since is may invalidate replays or whatever later.
  fee: 10_000n,
  nonce: await fetchNonce({ address: user.stxAddress, network }),
  network,
});

const signed = signTransaction(tx, user.stxPrivateKey);
await broadcastTransaction({ transaction: signed, network });
```

COMMENTS

what does " Pass `0n` / `0` to skip a
dimension." mean?
⎿ ⧉ Selected 2 lines from flows/2.md in Cursor

⏺ It refers to the unified buildStakeUpdate — it can do three things at once (extend
cycles, increase amount, rotate signer-manager), and each is opt-in. To skip one, you
pass a no-op value:

- cyclesToExtend: 0 → don't extend
- amountIncrease: 0n → don't add STX
- signerManager: <current value> → don't rotate

So "skip a dimension" = "don't do that one of the three things." E.g. just rotating:
cyclesToExtend: 0, amountIncrease: 0n, signerManager: newOne.

Worth flagging: this is awkward. Two alternatives:

1. Make them optional (cyclesToExtend?: number, omit to skip).
2. Split into three builders (buildStakeExtend, buildStakeIncrease,
   buildStakeRotateSigner) — but the contract is a single stake-update call, so this would
   just be three thin wrappers that all call the same thing.

Option 1 reads better. Want me to switch the flow to that shape?
