# Flow 6 — Grant a signer-manager permission to use a signer key

One-time setup before staking. The signer-key holder produces a SIP-018
signature authorizing a specific signer-manager contract; anyone can then
record that grant on-chain. After the grant lands, the signer-manager can
call `register-signer` to bind itself.

```ts
import { buildGrantSignerKey, signSignerKeyGrant } from '@stacks/bitcoin-staking';
import { broadcastTransaction, fetchNonce, signTransaction } from '@stacks/transactions';

const network = 'mainnet';
const signerManager = 'SP000…USER.signer-manager';
const authId = 1n;

// 1. Off-chain: signer-key holder signs (signerManager, authId) under the
//    pox-5-signer SIP-018 domain. Pure, no I/O.
const signerSignature = signSignerKeyGrant({
  signerManager,
  authId,
  network,
  privateKey: signer.privateKey,
});

// 2. On-chain: record the grant. Anyone can submit; replay is gated by
//    (signerKey, signerManager, authId).
const tx = buildGrantSignerKey({
  signerKey: signer.publicKey,            // 33-byte compressed pubkey, hex
  signerManager,
  authId,
  signerSignature,
  publicKey: user.stxPublicKey,
  fee: 10_000n,
  nonce: await fetchNonce({ address: user.stxAddress, network }),
  network,
});

const signed = signTransaction(tx, user.stxPrivateKey);
await broadcastTransaction({ transaction: signed, network });
```

After this lands, the signer-manager contract calls `pox-5::register-signer`
internally (tx-sender must be the manager principal); that step lives
inside the manager contract, not the SDK.
