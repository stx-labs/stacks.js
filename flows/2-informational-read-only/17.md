# Flow 17 — Browse an upcoming bond

A partner or pool operator inspects the next bond's parameters before
enrolling: capacity, target APY, ratio, open burn height, and their
own allowlist entry.

todo: add some more graphical elements to this file, to make clear what's important and what's not. for the ui.

```ts
import {
  fetchBond,
  fetchBondAllowance,
  fetchPoxInfo,
  fetchTotalSatsStakedForBond,
} from '@stacks/bitcoin-staking';

const network = 'mainnet';
const address = user.stxAddress;
const bondIndex = 7;

const [pox, bond, filledSats, allowance] = await Promise.all([
  fetchPoxInfo({ network }),
  fetchBond({ bondIndex, network }),
  fetchTotalSatsStakedForBond({ bondIndex, network }),
  fetchBondAllowance({ bondIndex, address, network }),
]);

const summary = {
  bondIndex,
  status: bond.openBurnHeight > pox.currentBurnchainBlockHeight ? 'announced' : 'open',
  openBurnHeight: bond.openBurnHeight,
  blocksUntilOpen: Math.max(0, bond.openBurnHeight - pox.currentBurnchainBlockHeight),
  capacitySats: bond.capacitySats,
  filledSats,
  remainingSats: bond.capacitySats - filledSats,
  targetRateBps: bond.targetRateBps,
  stxValueRatio: bond.stxValueRatio,
  minUstxRatioBps: bond.minUstxRatioBps,
  myAllocationSats: allowance,
};
```
