# Flow 23 — Bond calendar / schedule helper

Convert between bond index, reward cycle, and burn height. Useful for
notification timing, "X days until next bond opens," "in re-lock
window," and indexer pagination. The cycle/burn-height conversions are
pure math over `PoxInfo` (no contract round-trip); only the bond
existence check requires a fetch.

```ts
import {
  bondPeriodToBurnHeight,
  bondPeriodToRewardCycle,
  fetchFirstPox5RewardCycle,
  fetchPoxInfo,
  isBondActiveAtHeight,
  isInPreparePhase,
  rewardCycleToBurnHeight,
} from '@stacks/bitcoin-staking';

const network = 'mainnet';
const bondIndex = 7;

const [pox, firstBondPeriodCycleBig] = await Promise.all([
  fetchPoxInfo({ network }),
  fetchFirstPox5RewardCycle({ network }), // == first-bond-period-cycle on-chain
]);
const firstBondPeriodCycle = Number(firstBondPeriodCycleBig);

const openBurnHt = bondPeriodToBurnHeight({ bondIndex, firstBondPeriodCycle, poxInfo: pox });
const firstRewardCycle = bondPeriodToRewardCycle({ bondIndex, firstBondPeriodCycle });
const closeBurnHt = rewardCycleToBurnHeight({
  cycle: firstRewardCycle + 12, // BOND_LENGTH_CYCLES
  poxInfo: pox,
});

const reLockWindowBlocks = Math.floor(pox.rewardCycleLength / 2); // mainnet 1,050 (≈ 7.3d)

const now = pox.currentBurnchainBlockHeight;
const inPrepare = isInPreparePhase({ burnHeight: now, poxInfo: pox });
const isActiveNow = isBondActiveAtHeight({
  bondIndex,
  burnHeight: now,
  firstBondPeriodCycle,
  poxInfo: pox,
});

const schedule = {
  bondIndex,
  status:
    now < openBurnHt
      ? 'announced'
      : now < closeBurnHt - reLockWindowBlocks
        ? 'active'
        : now < closeBurnHt
          ? 're-lock-window'
          : 'closed',
  blocksUntilOpen: Math.max(0, openBurnHt - now),
  blocksUntilClose: Math.max(0, closeBurnHt - now),
  inPreparePhase: inPrepare,
  isActiveNow,
  openBurnHeight: openBurnHt,
  closeBurnHeight: closeBurnHt,
};
```

The re-lock window is canonically `rewardCycleLength / 2` (mainnet
1,050 blocks ≈ 7.3 days at the end of every bond, BTC unlocks before
STX). `inPrepare` is the gate flow 3's `unstake` and most update flows
hit.

`firstBondPeriodCycle` is read via `fetchFirstPox5RewardCycle` —
per `pox-5.clar:312-314` the contract's `first-pox-5-reward-cycle`
data-var is also the first bond period cycle.

## Lifecycle variants

This flow exists to **map heights to a status label** — lifecycle is
the whole point. Inputs (`openBurnHt`, `closeBurnHt`, `now`) never
change for a given bond; what moves is `now`. All variants below come
from the same fetch — only the derived `status` and the
`blocksUntilOpen` / `blocksUntilClose` deltas differ.

| `now` relative to bond | Derived status | Notable derived fields |
| --- | --- | --- |
| Pre-D-30 | (bond doesn't exist — `bondPeriodToBurnHeight` still returns a height, but the bond hasn't been declared on-chain; pair with `fetchBond` for a real existence check) | `blocksUntilOpen` very large; `isActiveNow: false`. |
| D-30 → D0 (`now < openBurnHt`) | `announced` | `blocksUntilOpen > 0`, `blocksUntilClose > 0`, `isActiveNow: false`. |
| D0 → D172 (`now ≥ openBurnHt` and `now < closeBurnHt - reLockWindowBlocks`) | `active` | `blocksUntilOpen: 0`, `isActiveNow: true`. |
| D172 → D182 (`now` within `reLockWindowBlocks` of `closeBurnHt`) | `re-lock-window` | `blocksUntilClose` ≤ ~1,050 (mainnet). BTC is unlocked on L1 here; `l1UnlockHeight` from `fetchBondL1UnlockHeight` should be ≤ `now`. STX still locked. |
| D183+ (`now ≥ closeBurnHt`) | `closed` | `blocksUntilClose: 0`, `isActiveNow: false`. Bond record stays readable; this flow only emits the label. |

`inPreparePhase` is orthogonal to bond D-days — it cycles every reward
cycle (~14 days) regardless of where any bond sits in its arc. Same for
`isInPreparePhase`-gated update flows (3, etc.).

The re-lock boundary uses `pox.rewardCycleLength / 2` per the
read-only docs (mainnet 1,050 blocks ≈ 7.3 days, testnet 525).
