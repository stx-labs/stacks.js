# Flow 18 — Quote required STX for a BTC commitment

Before enrolling, a partner or wallet UI computes how much STX must be
paired with a given sats amount under the bond's static ratio. Pure-math
mirror of the contract's `min-ustx-for-sats-amount` read so client and
contract stay bit-for-bit identical (bigint truncating division matches
Clarity uint division).

```ts
import { fetchBond, minUstxForSatsAmount } from '@stacks/bitcoin-staking';

const network = 'mainnet';
const bondIndex = 7;
const intendedSats = 2_500_000_000n; // 25 BTC

const bond = await fetchBond({ bondIndex, network });
if (!bond) throw new Error('bond not configured');

const requiredUstx = minUstxForSatsAmount({
  sats: intendedSats,
  stxValueRatio: bond.stxValueRatio, // ustx per 100 sats
  minUstxRatioBps: bond.minUstxRatioBps, // 5% min STX:BTC
});

// Sanity-display values for the UI.
const quote = {
  intendedSats,
  requiredUstx,
  requiredStx: Number(requiredUstx) / 1e6,
  ratioBps: bond.minUstxRatioBps,
};
```

`stxValueRatio` and `minUstxRatioBps` come from the bond record (set by
the Endowment in flow 4). Formula:
`(stxValueRatio * sats / 100) * minUstxRatioBps / 10000`.

## Lifecycle variants

Bond-agnostic, pure-math, no lifecycle dependence on the bond's D-day —
the ratio fields are set once at `setup-bond` (D-30) and are immutable
for the bond's life. The quote is identical at D-7, D0, D90, D177, and
D183 for a given bond.

The only D-day sensitivity is the upstream `fetchBond`:

| Phase | What changes for the quote |
| --- | --- |
| Pre-D-30 | `fetchBond` returns `undefined`; the snippet throws `'bond not configured'`. No quote possible. |
| D-30 onwards (forever) | `stxValueRatio` and `minUstxRatioBps` are stable for the bond's full life and remain readable after D183. The pure-math quote is meaningful at any point — but only actionable during the D-7 → D0 registration window. |

Note: `stxValueRatio` is described as "trailing-avg" but is fixed at
`setup-bond`-time on the `protocol-bonds` entry — it does not move
during the bond. If the contract evolves to update it mid-bond, this
flow becomes lifecycle-sensitive.
