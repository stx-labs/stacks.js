# Flow 14 — Watchdog reports a spent L1 UTXO

Anyone can post a Bitcoin proof that a tracked L1 lockup has been spent
before its CLTV expiry. The first valid proof earns compensation; the
reported position is dropped from T1 eligibility at the next payout.

The contract surface (`report-utxo-spent` or similar) is not yet
finalized in `pox-5.clar`; this is a sketch of the SDK shape.

```ts
import {
  buildReportUtxoSpent, // [MISSING surface — sketch]
  collectSpendProof, // [MISSING surface — sketch]
  fetchLockStatus, // [MISSING surface — sketch]
} from '@stacks/bitcoin-staking';
import { broadcastTransaction, fetchNonce, signTransaction } from '@stacks/transactions';

const network = 'mainnet';
const target = {
  staker: 'SP000…VICTIM',
  lockTxid: '…', // the original L1 lockup txid
  lockVout: 0,
};

// 1. Pull the Bitcoin proof: SPV-style merkle branch + spending tx.
const proof = await collectSpendProof({
  btcRpcUrl: 'https://mempool.space/api',
  lockTxid: target.lockTxid,
  lockVout: target.lockVout,
}); // Note: spend-proof collection is intentionally out of SDK scope —
   //       callers wire their own BTC node / explorer integration.

// 2. Submit the proof to L2. First valid submission earns compensation.
const tx = buildReportUtxoSpent({
  staker: target.staker,
  spendTxid: proof.spendTxid,
  spendBlockHeight: proof.blockHeight,
  merkleBranch: proof.merkleBranch,
  publicKey: watchdog.stxPublicKey,
  fee: 10_000n,
  nonce: await fetchNonce({ address: watchdog.stxAddress, network }),
  network,
});
await broadcastTransaction({ transaction: signTransaction(tx, watchdog.stxPrivateKey), network });

// 3. End-user read: 'locked' | 'spent-reported' | 'expired'.
const status = await fetchLockStatus({ address: target.staker, network });
```

## Lifecycle variants

**Active window:** D0 → D172 (only meaningful while the L1 CLTV has
*not* matured — after D172 spending the UTXO is the expected behavior,
not a watchdog signal).

| Phase | Behavior |
| --- | --- |
| D-7 → D0 | Pre-lock. There is no UTXO to report yet; allowlist entries and pending registrations are tracked separately. |
| D0 → D172 | Watchdog window. A spent UTXO here is a genuine fault: the staker bypassed the timelock via collusion with an early-exit signer outside the L2 announcement path, or the original lockup was never made. First valid proof earns compensation; reported position is dropped from T1 eligibility at the next payout. |
| D172 → D182 | CLTV matured — spending is legitimate (renewal via flow 11, or withdrawal). Watchdog reports are not meaningful here. |
| D183+ | Bond closed. Historical only. |

Cross-flow: flow 9 is the read-only counterpart that surfaces
`stillLocked = false` pre-D172; this flow is what turns that observation
into an on-chain action. Contract surface is not finalized — see open
question above; lifecycle gates here are inferred from the design, not
yet enforced by `pox-5.clar`.
