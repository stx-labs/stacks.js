# Flow 9 — Verify the L1 BTC lock for a paired bond

Given a bond membership on L2, reconstruct the deterministic P2WSH
address, then check Bitcoin to confirm the UTXO is still unspent and
report when it unlocks. The locking script is purely a function of the
staker's STX address, the bond's unlock height, and the bond's
early-unlock signer descriptor — so anyone can rederive it.

```ts
import {
  buildLockingBitcoinAddress,
  computeBondUnlockHeight,
  fetchBond,
  fetchBondMembership,
  fetchPoxInfo,
  fetchStakerSharesStakedByBond,
  firstPox5RewardCycle,
} from '@stacks/bitcoin-staking';

const network = 'mainnet';
const address = user.stxAddress;

const [pox, membership] = await Promise.all([
  fetchPoxInfo({ network }),
  fetchBondMembership({ address, network }),
]);
if (!membership) throw new Error('no paired bond for this address');
const firstBondPeriodCycle = firstPox5RewardCycle(pox);
if (firstBondPeriodCycle === undefined) throw new Error('pox-5 not yet activated');
// membership: { bondIndex, amountUstx, signer, isL1Lock }
// Note: amount-sats is no longer carried on the membership row; it is
// derived per-cycle from `get-staker-shares-staked-for-cycle`.

const bond = await fetchBond({ bondIndex: membership.bondIndex, network });

// Current locked sats for this (staker, signer, bond) leg. Used as the
// expected UTXO value below.
const lockedSats = await fetchStakerSharesStakedByBond({
  staker: address,
  signer: membership.signer,
  bondIndex: membership.bondIndex,
  network,
});

const unlockHeight = computeBondUnlockHeight({
  bondIndex: membership.bondIndex,
  firstBondPeriodCycle,
  poxInfo: pox,
});
const lockingAddress = buildLockingBitcoinAddress({
  stxAddress: address,
  unlockHeight,
  publicKey: user.btcPublicKey,
  earlyUnlockBytes: bond.earlyUnlockSigners,
  network: 'mainnet',
});

// Out-of-SDK: ask any Bitcoin explorer / node whether this address still
// holds an unspent output of the expected size.
const utxos = await fetch(`https://mempool.space/api/address/${lockingAddress}/utxo`).then(r =>
  r.json()
);
const stillLocked = utxos.some((u: any) => BigInt(u.value) === lockedSats);

const status = {
  lockingAddress,
  stillLocked,
  unlockBurnHeight: unlockHeight,
  blocksUntilUnlock: Math.max(0, unlockHeight - pox.currentBurnchainBlockHeight),
};
```

## Lifecycle variants

Read-only. Always callable, but `stillLocked` only carries information
after the CLTV matures at D172 — before that, the BTC *cannot* be spent
from the timelock branch, so the answer is trivially `true`.

| Phase | `stillLocked` semantics |
| --- | --- |
| D-7 → D0 | Pre-lock: the UTXO may not yet exist. `utxos` is empty → `stillLocked = false`. Distinguish "deposit not yet confirmed" from "spent" using `pox.currentBurnchainBlockHeight < unlockHeight` (true) plus an empty UTXO set. |
| D0 → D172 | Lock active. `stillLocked = true` is the expected state. A `false` here means an early-exit branch was exercised or the L1 lockup was never made — the position is no longer T1-eligible and a watchdog report (flow 14) may already be in flight. |
| D172 → D182 | Re-lock window. CLTV has matured; the UTXO can legitimately be spent into a renewal (flow 11) or withdrawn. `stillLocked = false` is normal here and not anomalous. `blocksUntilUnlock` clamps to 0. |
| D183+ | Bond closed. STX has unlocked on L2; the L1 side is whatever the user did with it. Historical read only. |

Cross-flow: this is the read counterpart to flow 14 (watchdog report).
Pre-D172, a `false` reading is what flow 14 turns into an on-chain
report; post-D172, the same value is benign.
