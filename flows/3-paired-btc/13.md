# Flow 13 — Early exit from a paired bond

The participant forfeits remaining BTC yield to retrieve their BTC
before the bond's natural unlock. Paired STX stays locked until the
bond ends and earns nothing for the remainder.

In the current contract there is **no staker-initiated L2 function**.
The participant's request is off-chain (to the Early Exit Coordinator);
the L2 cleanup is performed by the bond's `early-unlock-admin` via
`announce-l1-early-exit`. The end-to-end sequence:

1. **Participant** → HTTP request to the off-chain Early Exit
   Coordinator (a 1-of-N AWS multisig signer set). Status surface:
   `requested → co-signed → broadcast → confirmed`.
2. **Coordinator** co-signs an L1 spend against the pre-authorized
   early-exit branch (baked into the lockup script at enrollment — see
   `buildLockingBitcoinAddress` / `buildLockingScript` in flow 5: the
   `earlyUnlockBytes` push is sourced from `bond.earlyUnlockSigners`).
3. **L1 spend broadcast** to Bitcoin (the Coordinator or anyone).
4. **Admin** (the bond's `early-unlock-admin`) calls
   `announce-l1-early-exit(staker, oldSignerManager)` on L2 to settle
   rewards, zero the staker's bond shares, and decrement signer + total
   share counters. Only valid when the membership has `is-l1-lock =
   true`; sBTC positions use `unstake-sbtc` instead (flow 10).

```ts
// ────────────────────────────────────────────────────────────────
// Participant side — off-chain request to the Coordinator
// ────────────────────────────────────────────────────────────────

// The Coordinator HTTP surface is operated alongside the protocol; the
// SDK's `fetchEarlyExitStatus` wrapper is reserved (see fetch.ts) but
// not yet implemented. Until then, callers hit the Coordinator directly.

const request = await earlyExitCoordinator.requestEarlyExit({
  staker: user.stxAddress,
  bondIndex: bond.index,
});
// → { state: 'requested', requestId, ... }

// Poll or subscribe until state advances to 'broadcast' or 'confirmed':
const status = await earlyExitCoordinator.fetchStatus({ requestId: request.requestId });
// → { state: 'co-signed' | 'broadcast' | 'confirmed', btcTxid?, ... }

// ────────────────────────────────────────────────────────────────
// Admin side — L2 cleanup after the L1 spend confirms
// ────────────────────────────────────────────────────────────────
//
// Sent by the principal listed as `early-unlock-admin` on the bond
// (see Bond.earlyUnlockAdmin / `get-protocol-bond`). Must be a direct
// tx (no contract-caller indirection).

import { buildAnnounceL1EarlyExit } from '@stacks/bitcoin-staking';
import { broadcastTransaction, fetchNonce, signTransaction } from '@stacks/transactions';

const network = 'mainnet';

const announceTx = await buildAnnounceL1EarlyExit({
  staker: user.stxAddress,
  oldSignerManager: 'SP000…USER.signer-manager', // must match the staker's current signer
  publicKey: admin.stxPublicKey,
  fee: 10_000n,
  nonce: await fetchNonce({ address: admin.stxAddress, network }),
  network,
});
const signed = signTransaction(announceTx, admin.stxPrivateKey);
await broadcastTransaction({ transaction: signed, network });

// Effect: settles outstanding rewards for the (signer, bond) leg,
// zeroes the staker's bond shares, decrements the signer's total
// shares and the bond's total share counter. The participant's
// paired STX remains locked through D182.
```

**Design intent vs. current contract.** The launch-scope and
user-flows references envision a staker-initiated `request-early-exit`
L2 function that would flag the position itself. The current contract
ships only the admin-side `announce-l1-early-exit` for L2 cleanup; the
participant's intent is conveyed off-chain to the Coordinator. If a
participant-initiated L2 surface is added later, the request side of
this flow will collapse onto it; the admin cleanup step stays.

Open: SDK wrapper for the Coordinator HTTP API
(`fetchEarlyExitStatus` is reserved in `fetch.ts` but not implemented);
whether the SDK ships a thin Coordinator client or only documents the
contract.

## Lifecycle variants

**Active window:** D0 → D172 (while the bond is active and the L1
CLTV hasn't matured). After D172 the natural unlock path is available,
so early exit is moot.

| Phase | Behavior |
| --- | --- |
| D-7 → D0 | Bond hasn't started — `announce-l1-early-exit` reverts with `ERR_NOT_BOND_PARTICIPANT` (no membership row yet for the staker). |
| D0 → D172 | Meaningful window. Participant requests off-chain → Coordinator co-signs the pre-authorized L1 spend → admin calls `announce-l1-early-exit` on L2 to drop the position from T1 eligibility. Paired STX stays locked through D182. |
| D172 → D182 | Re-lock window. CLTV has matured; the staker can spend on L1 unilaterally without the early-exit branch. Calling `announce-l1-early-exit` is still valid but redundant — the natural unlock has the same effect. |
| D183+ | Bond closed. No-op territory. |
