# Flow 13 — Early exit from a paired bond

The user forfeits remaining BTC yield to retrieve their BTC before the
bond's natural unlock. Paired STX stays locked until the bond ends and
earns nothing. Three actors per the latest design (`staking.stacks.co —
User Flows & App Architecture`, §3 architecture diagram):

1. **L2 (`pox-5`)** exposes `request-early-exit`. Submitting it flags
   the position and stops T1 yield accrual.
2. **Early Exit Coordinator** — off-chain service (1-of-N AWS multisig).
   Watches L2 for the request, co-signs an L1 spend against the
   pre-authorized branch, exposes
   `requested → co-signed → broadcast → confirmed` status.
3. **L1 (Bitcoin)** — anyone broadcasts the co-signed spend. The
   early-exit branch must already be baked into the L1 unlock script at
   enrollment (flag on `buildLockingScript`, see flow 5/9).

```ts
import {
  buildEarlyExitRequest, // [MISSING surface — sketch]
  fetchEarlyExitStatus, // [MISSING surface — sketch]
} from '@stacks/bitcoin-staking';
import { broadcastTransaction, fetchNonce, signTransaction } from '@stacks/transactions';

const network = 'mainnet';

// 1. L2: flag the position for forfeit. Stops future T1 payouts.
const exitTx = buildEarlyExitRequest({
  publicKey: user.stxPublicKey,
  fee: 10_000n,
  nonce: await fetchNonce({ address: user.stxAddress, network }),
  network,
});
await broadcastTransaction({ transaction: signTransaction(exitTx, user.stxPrivateKey), network });

// 2. The Early Exit signer service observes the event and returns a
//    co-signed BTC spend ready to broadcast (or broadcasts itself).
const status = await fetchEarlyExitStatus({ address: user.stxAddress, network });
const cosignedBtcTx = await earlyExitSignerClient.fetchCosignedSpend({
  staker: user.stxAddress,
});
// → broadcast cosignedBtcTx to Bitcoin (mempool.space, custom node, …)

// Paired STX remains locked; weekly payouts to this position have stopped.
```
