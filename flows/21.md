# Flow 21 — Pool operator monitor view

After flow 20, the operator wants a single read-only snapshot of how
their bond is performing on pox-5: total sats committed, share of the
bond, current cycle's claimable rewards, and rewards already paid.
Member-level accounting lives in the pool contract; this flow only
covers the pox-5 surface.

```ts
import {
  fetchClaimableRewards,
  fetchCurrentDistributionCycle,
  fetchPoxInfo,
  fetchSignerRewardsPaidForCycle,
  fetchSignerSharesStakedForCycle,
  fetchTotalSatsStakedForBond,
} from '@stacks/bitcoin-staking';

const network = 'mainnet';
const signerManager = 'SP000…COMMUNITY-POOL.v1';
const bondIndex = 7;

const [pox, distCycle, bondTotal, mySharesBond, paidThisCycle, claimable] = await Promise.all([
  fetchPoxInfo({ network }),
  fetchCurrentDistributionCycle({ network }),
  fetchTotalSatsStakedForBond({ bondIndex, network }),
  fetchSignerSharesStakedForCycle({ signerManager, index: bondIndex, isBond: true, network }),
  fetchSignerRewardsPaidForCycle({ signerManager, index: bondIndex, isBond: true, network }),
  fetchClaimableRewards({ signerManager, rewardCycle: 0, bondIndices: [bondIndex], network }),
]);

const dashboard = {
  cycle: pox.rewardCycleId,
  distributionCycle: distCycle,
  bondTotalSats: bondTotal,
  mySatsInBond: mySharesBond,
  myShareBps: bondTotal === 0n ? 0 : Number((mySharesBond * 10_000n) / bondTotal),
  rewardsPaidLifetime: paidThisCycle,
  claimableNow: claimable,
};
```

The `claimable` read is what the pool's `claim-rewards` (flow 7) will
pull on the next call. `myShareBps` is informational — pool-internal
member weights drive the actual onward distribution.
