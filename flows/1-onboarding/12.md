# Flow 12 â€” Delegate calls to a helper contract

The user authorizes another contract (e.g. a wallet's batching helper or
a UI's relayer) to make pox-5 calls on their behalf, optionally with an
expiry. Disallow at any time.

```ts
import {
  buildAllowContractCaller,
  buildDisallowContractCaller,
  fetchAllowanceContractCallers,
} from '@stacks/bitcoin-staking';
import { broadcastTransaction, fetchNonce, signTransaction } from '@stacks/transactions';

const network = 'mainnet';
const helper = 'SP000â€¦WALLET.batch-helper';

// 1. Allow until burn height 900_000 (omit untilBurnHeight for no expiry).
const allowTx = buildAllowContractCaller({
  contractCaller: helper,
  untilBurnHeight: 900_000,
  publicKey: user.stxPublicKey,
  fee: 10_000n,
  nonce: await fetchNonce({ address: user.stxAddress, network }),
  network,
});
await broadcastTransaction({ transaction: signTransaction(allowTx, user.stxPrivateKey), network });

// Anyone can verify the allowance is in effect.
const allowed = await fetchAllowanceContractCallers({
  sender: user.stxAddress,
  contractCaller: helper,
  network,
});
// â†’ { callerAllowed: boolean, callerExpiryHeight?: number }

// 2. Revoke when no longer needed.
const disallowTx = buildDisallowContractCaller({
  contractCaller: helper,
  publicKey: user.stxPublicKey,
  fee: 10_000n,
  nonce: await fetchNonce({ address: user.stxAddress, network }),
  network,
});
await broadcastTransaction({ transaction: signTransaction(disallowTx, user.stxPrivateKey), network });
```

## Lifecycle variants

**Bond-agnostic.** `allow-contract-caller` / `disallow-contract-caller` operate on `(sender, contract-caller)` â€” no bond index, no reward cycle. The grant lives on its own clock via `untilBurnHeight`, orthogonal to any bond's D-day.

A single user may hold callers covering positions across multiple concurrently-active bonds (a D-30 announcement, a D90 active bond, a D177 re-lock-window bond) â€” one allowance row authorizes all of them simultaneously, as long as the helper is what's submitting the on-chain calls.

| Phase | What's different |
| --- | --- |
| Pre-grant | Helper calls into pox-5 on the user's behalf revert with `ERR_UNAUTHORIZED_CALLER (u22)` from `check-caller-allowed`. |
| Grant active (`burn-block-height < untilBurnHeight` or no expiry) | `fetchAllowanceContractCallers â†’ { callerAllowed: true }`. Helper can drive every pox-5 flow the user qualifies for. |
| Grant expired (`burn-block-height >= untilBurnHeight`) | The map row may still exist but `check-caller-allowed` returns false; treat as revoked. `fetchAllowanceContractCallers` surfaces `callerExpiryHeight` for the UI to render countdown / "expired" state. |
| After `disallow` | Map entry deleted. Identical to pre-grant; re-allow with a new `allow-contract-caller` tx. |
