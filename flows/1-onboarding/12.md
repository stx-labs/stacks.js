# Flow 12 — Call pox-5 through a helper contract

A wallet's batching helper, a UI's relayer, or a custodial wrapper can submit
pox-5 calls for a user. pox-5 identifies the staker as `tx-sender`, so the
staker-facing entrypoints (`register-for-bond`, `update-bond-registration`,
`stake`, `stake-update`, `unstake`, `unstake-sbtc`) impose no contract-caller
gate — there is no pre-authorization step to manage.

The user signs the originating transaction (they are `tx-sender`); the wrapper
contract is the `contract-caller`. The wrapper must carry the appropriate
STX/sBTC post-conditions, since `tx-sender` is the principal whose assets move.
No SDK call is needed to enable this — build the underlying pox-5 transaction as
usual (e.g. `buildRegisterForBond`) and route it through the helper.

## Caller-constrained entrypoints

A few entrypoints still pin the caller and cannot be routed through a wrapper:

| Entrypoint | Constraint |
| --- | --- |
| `announce-l1-early-exit` | Staker must call directly — `contract-caller == tx-sender == staker`. |
| `claim-rewards` / `claim-staker-rewards-for-signer` | `contract-caller` is treated as the signer-manager principal. |
| `grant-signer-key` / `revoke-signer-grant` | `contract-caller == signer-manager`. |
| `setup-bond` / `set-bond-admin` | `contract-caller == bond-admin`. |
