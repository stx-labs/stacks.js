# Flow 4 — Endowment sets up a bond and seeds its allowlist

Admin-only. The bond admin (Endowment) configures a new bond period and
seeds its `(staker, max-sats)` allowlist in a single `setup-bond` call.
This must run before any partner can enroll (flow 5).

```ts
import { buildSetupBond } from '@stacks/bitcoin-staking';
import { broadcastTransaction, fetchNonce, signTransaction } from '@stacks/transactions';

const network = 'mainnet';

const tx = buildSetupBond({
  bondIndex: 7,
  targetRateBps: 500, // 5% target APY
  stxValueRatio: 1_200_000_000n, // ustx per 100 sats (trailing-avg STX/BTC)
  minUstxRatioBps: 500, // 5% minimum STX:BTC
  earlyUnlockSigners: earlyExitDescriptor, // 683-byte multisig descriptor
  allowlist: [
    { staker: 'SP000…PARTNER_A', maxSats: 2_500_000_000n }, // 25 BTC
    { staker: 'SP000…PARTNER_B', maxSats: 1_000_000_000n }, // 10 BTC
    { staker: 'SP000…POOL_OPERATOR', maxSats: 5_000_000_000n },
  ], // todo: maxSats should be able to take bignint or string for the value
  publicKey: endowment.stxPublicKey,
  fee: 10_000n,
  nonce: await fetchNonce({ address: endowment.stxAddress, network }),
  network,
});

const signed = signTransaction(tx, endowment.stxPrivateKey);
await broadcastTransaction({ transaction: signed, network });
```
