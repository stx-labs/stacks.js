# Flow 4 — Endowment sets up a bond and seeds its allowlist

Admin-only. The bond admin (Endowment) configures a new bond period and
seeds its `(staker, max-sats)` allowlist in a single `setup-bond` call.
This must run before any partner can enroll (flow 5).

```ts
import { buildSetupBond } from '@stacks/bitcoin-staking';
import { broadcastTransaction, fetchNonce, signTransaction } from '@stacks/transactions';

const network = 'mainnet';

const tx = buildSetupBond({
  bondIndex: 7,
  targetRateBps: 500, // 5% target APY
  stxValueRatio: 1_200_000_000n, // ustx per 100 sats (trailing-avg STX/BTC)
  minUstxRatioBps: 500, // 5% minimum STX:BTC
  earlyUnlockSigners: earlyExitDescriptor, // 683-byte multisig descriptor
  earlyUnlockAdmin: 'SP000…ENDOWMENT', // principal allowed to call announce-l1-early-exit
  allowlist: [
    { staker: 'SP000…PARTNER_A', maxSats: 2_500_000_000n }, // 25 BTC
    { staker: 'SP000…PARTNER_B', maxSats: 1_000_000_000n }, // 10 BTC
    { staker: 'SP000…POOL_OPERATOR', maxSats: 5_000_000_000n },
  ],
  publicKey: endowment.stxPublicKey,
  fee: 10_000n,
  nonce: await fetchNonce({ address: endowment.stxAddress, network }),
  network,
});

const signed = signTransaction(tx, endowment.stxPrivateKey);
await broadcastTransaction({ transaction: signed, network });
```

## Lifecycle variants

**Active window:** roughly **D-28 → D-1** — the contract requires `bond-start-height - BOND_GAP_CYCLES * pox-reward-cycle-length <= burn-block-height < bond-start-height` (`BOND_GAP_CYCLES = 2` reward cycles, ~28 days on mainnet). Outside this window the call reverts with `ERR_CANNOT_SETUP_BOND_TOO_SOON` (before) or `ERR_CANNOT_SETUP_BOND_TOO_LATE` (after). One-shot per bond — within the window, a second call reverts with `ERR_BOND_ALREADY_SETUP`, so the allowlist is **sealed at announce-time**; no add/remove later.

| D-day | What's different |
| --- | --- |
| D-28 → D-1 | Canonical window. Endowment publishes capacity, params, allowlist. `protocol-bonds` row created; `protocol-bond-allowances` seeded. Re-running for the same `bondIndex` reverts with `ERR_BOND_ALREADY_SETUP` (only checked once we're inside the window — outside, the too-soon / too-late asserts fire first). Use a fresh `bondIndex` to announce the *next* bond — multiple bonds run concurrently. |
| D0+ | Too late: `ERR_CANNOT_SETUP_BOND_TOO_LATE`. If an allowlist row is wrong, the staker simply can't register; no on-chain fix exists. |

Because six bonds run concurrently (announced every ~2 cycles), at any wall-clock moment one bond is at D-30 while others sit at D0 / D90 / D177. The admin runs this flow continuously against a rolling set of `bondIndex` values.
